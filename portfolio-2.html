CSC8470 - Cornell-Cards Application

Project Documentation

Project Overview

The Cornell-Cards application was developed as part of a collaborative project (CSC8470) which aimed at creating a notetaking and knowledge retention tool. The application is inspired by the Cornell notetaking method and is designed to help users organize study materials and review them using a spaced repetition study schedule. The backend application is designed to be paired with a (hypothetic) client-side interface, allowing users to manage notes organized into folders, create cards with concept-explanation pairs, and schedule study sessions. The project adopts an n-tier architectural pattern to promote a clean separation of concerns, ensuring a modular and maintainable codebase.

Implementation

The project involved the following key components:

  • N-Tier Architecture - The application is divided into three main layers:
    • Wgeb API Layer - which manages HTTP requests/responses, handles authentication, and provides API endpoints for CRUD operations on notes, folders, and cards.
    • Domain Layer - which contains the business logic, including entity models and services like the spaced repetition algorithm for rescheduling cards.
    • Infrastructure Layer - which manages data access, including repository implementations and EF Core for database interactions.
  • The core feature of the application is the spaced repetition algorithm, which schedules card reviews based on user ratings. The algorithm adjusts the review interval depending on the difficulty of recall.
  • Entity Framework Core (EF Core) was used for data access and manipulation, initially adopting a database-first approach and later integrating ASP.NET Identity for user authentication. The project incorporated automatic database instantiation and seeding for testing purposes.
  • To improve performance, the application uses asynchronous methods for database operations, particularly in the repository classes.
  • The application uses ASP.NET Identity and JSON Web Tokens (JWT) for secure user authentication and authorization. The ASP.NET middleware pipeline checks JWT validity and ensures users can only access their resources.
  • Git was used for version control, with a feature branch strategy to allow isolated development of features and integration into the main codebase.
  • The xUnit framework, along with Moq for mocking dependencies, was used for unit testing the applicationâ€™s components.
  • Serilog was configured for error logging, capturing and storing logs for analysis and troubleshooting. Logs were categorized by severity and stored in a local file sink.

Lessons

The Cornell-Cards project completed as part of CSC8470 provided a practical introduction to backend development. The most important takeaways include:

  1. Learning how to implement security measures in Web API projects using JSW, ASP.NET Identify and ASP.NET authentication and authorization middleware pipelines
  2. Introduction to scalable software architecture patterns including the n-tier architecture along with design patterns including repositories and services.
  3. How to incorporate unit testing with the xUnit framework in a practical project.
