Portfolio Project 1

Description

  • AKA Studios

  • 2024

This project, part of the CSC6200 capstone, developed an ICT solution using Microsoft365 tools to address workflow inefficiencies for a B2B service business. The implementation included a SharePoint Hub Site, a Power App interface, and automated workflows via Power Automate to streamline project management and document handling.

Project Overview

Initiated as part of the CSC6200 capstone unit, this project involved the development of an ICT solution for a real-world client as part of a professional team. Our client, a small business specializing in B2B services including marketing and brand development, faced workflow inefficiencies primarily due to a lack of project discoverability and efficient document management. The existing setup failed to effectively capture project information and project activity, leading to delays, miscommunication, and difficulties tracking progress.

To address these issues, our student team developed a comprehensive solution incorporating Microsoft 365 products, specifically SharePoint, Power App, and Power Automate. As project manager, I coordinated the project throughout three sprint periods:

  • Sprint 1: SharePoint implementation
  • Sprint 2: Development of Power App functionality
  • Sprint 3: Finalization and documentation

Implementation

The solution incorporated the following aspects:

  1. SharePoint Hub Site – The primary hub site was used for storage of project information (project details, client contact information, and project activity logs) as well as the project-specific documents and files.
  2. Power App – The user interface (UI) which used the information/files stored in SharePoint across six (6) screens:
    • Dashboard: Used to display a list of projects, filtered by status and stage. This screen also incorporates options for creating new clients and projects. Users can “open” a project to view project-specific screens (described below).
    • Overview: View and edit information of a specific project.
    • Client Details: View and edit client information.
    • Documents: In-app SharePoint folder navigation with options for viewing document details, file upload, check-out, and check-in (integrated with SharePoint versioning).
    • Activities: Displays the activity logs for that project which are automatically generated when users edit project details.
    • Settings: Troubleshooting information and options to refresh data.
  3. Power Automate – A number of Power Automate “flows” were developed to implement more complex workflows. These flows are triggered manually from user actions in Power App or when certain events occur. These workflows included log generation, project folder creation, file upload to SharePoint, document check-out, and check-in from SharePoint (integrating with its versioning capabilities).

Lessons

There were many important takeaways and adjustments to the way that I would personally approach future ICT projects. Of the many different lessons discussed in this self-reflection report, the three most significant changes will include:

  1. Rethink how I approach agile software development planning by implementing features based on their importance to the business rather than their association with a specific part of the technology stack. Furthermore, validate that the most critical features will work as expected by developing a prototype rather than just relying on research.
  2. Further develop the processes that I use during sprint planning and retrospectives. Rather than simply defining the tasks that need to be completed, incorporate holistic considerations such as quality requirements, budget implications, etc. This will improve communication with the project team and ensure better adherence to the project plan.
  3. Improve communication skills, including how I approach open-ended questions, as well as preparing/sharing meeting materials in advance. Practicing these skills in the future will help ensure that meetings are shorter while providing more value.
